# Sparsification (Zero-Skip) Analysis

Generated by `scripts/sparsification_analysis.sh`.

## What Is Zero-Skip?

When `zeroSkip=true`, a comparator gates the multiplier:
```
if a == 0 || b == 0: accOut = accIn   // bypass: no multiply
else:                 accOut = a*b + accIn
```
At 50% input sparsity (ReLU activations or pruned weights), roughly half of
all multiply-accumulate cycles are eliminated in time-multiplexed designs.

## Notes

- Mul-line proxy counts lines containing ` * ` in the generated SystemVerilog.
  For time-multiplexed designs (SerialSharedFabric, UnifiedSerial) this equals
  the number of physical multipliers, not the operation count.
- Skip estimate is ⌈mul_proxy / 2⌉ — a structural upper bound at 50% sparsity.
  Mul-line proxy counts physical multiplier lines in the generated SV; for
  time-multiplexed designs one multiplier covers many MAC cycles, so this is
  NOT a dynamic cycle-saved count. Actual savings depend on runtime sparsity.
- ZeroSkip adds a comparator (an OR gate on sign-extended inputs) and a Mux.
  The area overhead is one ~N-bit OR plus one ~N-bit Mux per MAC.
